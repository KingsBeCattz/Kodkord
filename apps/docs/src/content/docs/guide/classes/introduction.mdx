`@kodkord/classes` offers an even more robust experience, thanks to classes dedicated to each type of Discord API data.

<Aside type="caution">`@kodkord/classes` is completely independent of `kodkord`, since it is based on using the raw data from the Discord API.</Aside>

### Before Installation <Badge text="IMPORTANT" size="small" variant="caution" style={{ fontWeight: "bolder", textDecoration: "underline" }} />
We recommend first of all to have installed:
<div class="div-before-installation">
  <Icon name="right-caret" label=">" class="display-inline" />[<Icon name="node" class="display-inline" /> NodeJS](https://nodejs.org/) (v18 or higher)
</div>
<div class="div-before-installation">
  <Icon name="right-caret" label=">" class="display-inline" />[<Icon name="seti:typescript" class="display-inline" /> TypeScript](https://www.typescriptlang.org/) (v4.3 or higher) <Badge variant="tip" text="OPTIONAL USE"/>
</div>
<div class="div-before-installation">
  <Icon name="right-caret" label=">" class="display-inline" /> Any package manager, below are some examples
</div>

<Aside type="note">
- TypeScript is not required to use `@kodkord/classes` <tr/>
- `@kodkord/classes` was recently released, that means there may be bugs. Also, backwards compatibility with node and typescript has not been tested. So we would appreciate it if you could contribute by testing :3.
</Aside>

<Aside type="tip">
  In case you want to have the mirror version of Github installed, i.e. the development version, just replace `@kodkord/classes` with `@kodkord/classes@dev`, you can also have multiple instances, for example:
  
<Tabs syncKey="pkg">

    <Code code=" bun add @kodkord/classes-stable@npm:@kodkord/classes" lang="sh" frame="terminal" class="JetBrains-Mono-Semibold" title="Stable"/>
    <Code code=" bun add @kodkord/classes-dev@npm:@kodkord/classes@dev" lang="sh" frame="terminal" class="JetBrains-Mono-Semibold" title="Dev"/>
    <Aside type="caution">
      When installing an aliased dependency with bun, instead of saving it in your package.json with, for example: `"@kodkord/classes-dev": "npm:@kodkord/classes@0.0.2-dev-13076173580.0"`, save it as: `"@kodkord/classes-dev": "^0.0.2-dev-13076173580.0"`.

      This can cause problems with other package managers as they will look for `@kodkord/classes-dev` in the npm registry, you can fix this by replacing `â€œ^0.0.2-dev-13076173580.0"` with `"npm:@kodkord/classes@0.0.2-dev-13076173580.0"`.
    </Aside>
  </TabItem>
